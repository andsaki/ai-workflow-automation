# /resume-session

過去のセッション履歴を確認し、前回の作業内容を把握して続きを開始します。

## 実行手順

### 1. セッション一覧の表示

利用可能なセッション一覧を表示：

```
使用可能なセッションIDをリストアップし、最新の5セッションを表示
各セッションの最終更新時刻と概要を表示
```

### 2. 最新セッションの内容を確認

最新のセッションから以下の情報を抽出：

**作業内容の確認**:
- 最後に編集したファイル
- 実行していたタスク
- 未完了のTodoリスト
- 最後の会話内容

**Git状態の確認**:
```bash
!git status --short
```

現在のブランチ：
```bash
!git branch --show-current
```

最近のコミット：
```bash
!git log --oneline -n 5
```

### 2.5. /tmp/ 内のバックアップファイルを確認

作業中に保存されたバックアップファイルや実験的なコードを確認：

**/tmp/ 内の開発関連ファイルを探す**:
```bash
!ls -lt /tmp/*.{tsx,ts,vue,css,js,py,jsx,json} 2>/dev/null | head -10
```

**最近変更されたファイルを詳細に確認**:
```bash
!find /tmp -maxdepth 1 -type f \( -name "*.tsx" -o -name "*.ts" -o -name "*.vue" -o -name "*.css" -o -name "*.js" -o -name "*.py" -o -name "*.jsx" \) -mtime -7 -exec ls -lh {} \; 2>/dev/null
```

**バックアップファイルの内容確認**:
- css-version.tsx などの命名パターンを探す
- プロジェクト名や機能名が含まれるファイル
- revert や backup といったキーワードを含むファイル

見つかったファイルは Read ツールで内容を確認し、以下を判断：
- 前回の作業で作成されたバックアップか
- revert された実装の保存版か
- 実験的な実装の試行版か
- 復元が必要なコードか

### 3. 作業コンテキストの復元

前回のセッションで行っていた作業を要約：

**確認項目**:
1. 実装中だった機能
2. 修正中だったバグ
3. 未完了のタスク
4. 次に行う予定だった作業

**関連ファイルの確認**:
最後に編集したファイルがあれば、その内容を確認：

```bash
!git diff HEAD~1 HEAD --name-only
```

### 4. 作業の再開提案

前回のセッション内容を基に、以下を提案：

**再開オプション**:
1. **前回の続きを開始** - 未完了タスクから再開
2. **新しいタスクを開始** - 前回とは別の作業を開始
3. **前回の作業をレビュー** - 変更内容を確認してから判断

**提案例**:
```
前回のセッションでは「Panda CSSのデザインシステム実装」を行っていました。

未完了のタスク:
- [ ] Buttonコンポーネントの実装
- [ ] Cardコンポーネントのレスポンシブ対応

前回の最後に編集したファイル:
- components/design-system/Button.vue
- panda-config/recipes/button.ts

/tmp/ 内のバックアップファイル:
- /tmp/css-version.tsx (CSS実装版のバックアップ - revert済み)

続きから始めますか？それとも別の作業を始めますか？
```

### 5. 次のアクションを確認

ユーザーに次のアクションを確認：

1. 前回の続きを再開する
2. 別の作業を開始する
3. 変更内容をレビューしてから判断する
4. Git状態を整理してから再開する

## 分析のポイント

### セッション履歴から読み取る情報

**ファイル編集履歴**:
- 最後に編集したファイルの種類
- 複数ファイルにまたがる変更か
- 新規ファイルの作成か既存ファイルの編集か

**タスクの進捗**:
- Todoリストの状態
- 完了したタスク数
- 残っているタスク数

**会話の流れ**:
- 最後にユーザーが依頼した内容
- AIが提案した次のステップ
- 実装中のエラーや課題

**Git状態**:
- ステージングされた変更
- コミットされていない変更
- 現在のブランチ

**バックアップファイル**:
- /tmp/ 内の開発関連ファイル
- revert された実装の保存版
- 実験的なコードの試行版
- 代替実装のバックアップ

### 作業の優先順位

1. **未完了タスクが明確** → すぐに再開
2. **エラーで中断** → エラー解決から再開
3. **レビュー待ち** → 変更内容を確認
4. **コミット前** → コミットしてから次へ

---

## 使用例

```
/resume-session
```

または自然な会話で：
- 前回の続きをやりたい
- セッション履歴を見せて
- 何やってたっけ？
- 続きから始めたい

---

## セッション情報の活用

### 効率的な作業再開のために

**前回のコンテキストを活用**:
- 同じファイルを編集していた → 続きから編集
- 特定の機能を実装中 → 同じ方針で継続
- ライブラリを調査中 → 前回の調査結果を参照

**Git状態に応じた対応**:
- 変更がコミットされている → 新しい作業を開始
- 変更がステージングされている → コミットしてから再開
- 変更がステージングされていない → 内容を確認してステージング

**作業の連続性を保つ**:
- デザインシステムの実装途中 → 同じコンポーネントから再開
- バグ修正中 → テストを実行して状態確認
- ドキュメント作成中 → 同じセクションから継続

**バックアップファイルの活用**:
- revert された実装 → 必要なら復元や参照
- 実験的なコード → 別アプローチの検討材料
- 代替実装の保存 → 比較やマージの参考

---

## 注意事項

- セッション履歴は自動的に保存されます
- 過去のセッションは参照のみで編集できません
- 作業を再開する前に、必ずGit状態を確認してください
- 前回の変更がコミットされていない場合は注意が必要です
- セッション間で環境が変わっている可能性があります（依存関係の更新など）

## トラブルシューティング

### セッション情報が見つからない場合
```
セッション履歴が見つかりません。
新しい作業を開始しますか？
```

### Git状態が前回と大きく異なる場合
```
前回のセッション以降、複数のコミットが追加されています。
変更内容を確認してから作業を再開することをお勧めします。
```

### 前回のセッションでエラーが発生していた場合
```
前回のセッションではエラーで中断しています：
[エラー内容]

このエラーを解決してから作業を再開しますか？
```

### /tmp/ 内にバックアップファイルが見つかった場合
```
/tmp/ 内に以下のバックアップファイルが見つかりました：
- /tmp/css-version.tsx (2026-01-30 12:06)

このファイルは前回の作業で保存された代替実装です。
内容を確認してプロジェクトに反映しますか？
```
