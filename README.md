# AI Workflow Automation

Claude Code と Codex CLI で使える Git ワークフロー自動化スキル/プロンプトの管理リポジトリです。

## 🎯 このリポジトリについて

このリポジトリは **AIスキルとプロンプトを管理するための専用リポジトリ** です。

### 重要な設計思想

- ✅ このリポジトリには **AIスキルの定義ファイル** のみを管理します
- ✅ スキルを実行すると、**開発リポジトリ** にワークフローファイルやコードを生成します
- ❌ このリポジトリ自体に `.github/workflows/` や実装コードは置きません

### 具体例

例えば `/setup-auto-close-pr` スキルの場合：

```
このリポジトリ (ai-workflow-automation):
  └── claude/commands/setup-auto-close-pr.md  ← スキル定義のみ

開発リポジトリ (your-project):
  └── .github/workflows/auto-close-linked-pr.yml  ← スキル実行で生成される
```

このアプローチにより、スキルを一箇所で管理し、複数の開発リポジトリで再利用できます。

## スキル一覧

### 🔧 基本 Git スキル

| スキル | 説明 |
|-------|------|
| `/commit` | AIがコミットメッセージを生成してコミット |
| `/push` | 現在のブランチをリモートにプッシュ |
| `/commit-push` | コミット＋プッシュを一括実行 |
| `/create-pr` | AIがPRタイトル・本文を生成してPR作成 |
| `/branch-name-helper` | 変更内容から適切なブランチ名を提案・作成 |
| `/resume-session` | 前回のセッションを再開 |

### 🚀 Issue ワークフロー

| スキル | 説明 |
|-------|------|
| `/implement-issue <番号>` | Issueを読んで実装（各ステップで確認） |
| `/auto-implement-issue <番号>` | Issue完全自動実装 |
| `/close-issue <番号>` | Issueをコメント付きでクローズ |

### 🏗️ プロジェクト管理

| スキル | 説明 |
|-------|------|
| `/start-impl` | 新機能の実装セッションを開始 |
| `/update-ai-guide` | セッション内容からai-guideを自動更新 |

### ⚙️ GitHub Actions

| スキル | 説明 |
|-------|------|
| `/setup-auto-close-pr` | PRがApproveされたら連動PRを自動クローズするワークフローを設定 |

## 基本的な Git ワークフロー

### パターン1: シンプルな変更

```
1. コードを編集
2. "変更をコミットして" または /commit
3. "プッシュして" または /push
```

### パターン2: コミット＋プッシュを一気に

```
1. コードを編集
2. /commit-push
```

実行内容：
- ✅ 型チェック
- ✅ Lint実行
- ✅ AIがコミットメッセージ生成
- ✅ コミット
- ✅ プッシュ

### パターン3: PR作成まで

```
1. /branch-name-helper（ブランチ名提案）
2. コードを編集
3. /commit-push
4. /create-pr
```

## Issue ベースのワークフロー

### パターン1: 手動確認しながら実装

```
/implement-issue 123
```

または自然な会話で：

```
Issue #123 を実装して
```

**実行フロー**:
1. 🔍 Issue読み取り
2. 🌿 ブランチ作成（提案→確認）
3. 📝 実装計画提示（**ユーザー承認待ち**）
4. ⚙️ 実装
5. 🧪 テスト・ビルド
6. 💾 コミット（メッセージ確認）
7. 📤 プッシュ
8. 🔀 PR作成（タイトル・本文確認）
9. ✅ Issueコメント

**推奨用途**: 重要な変更、大規模な実装

### パターン2: 完全自動実装

```
/auto-implement-issue 123
```

または自然な会話で：

```
Issue #123 を完全自動で実装して
```

**実行フロー**:
1. 🔍 Issue読み取り
2. 🌿 ブランチ自動作成
3. ⚙️ 自動実装
4. 🧪 テスト・ビルド
5. 🔧 エラー自動修正（最大3回）
6. 💾 自動コミット
7. 📤 自動プッシュ
8. 🔀 自動PR作成
9. ✅ Issueコメント

**推奨用途**: 小〜中規模の変更、単純なバグ修正

⚠️ **注意**: 確認なしで全自動実行します。重要な変更は `/implement-issue` を使用してください。

### パターン3: Issue完了

```
/close-issue 123
```

オプション：
```
/close-issue 123 not_planned
```

**注意**: 通常はPRマージ時に自動クローズされるため、このスキルは不要です。

## スキル比較表

### Git基本操作

| やりたいこと | スキル | 所要時間 |
|------------|-------|---------|
| コミットだけ | `/commit` | 30秒 |
| プッシュだけ | `/push` | 10秒 |
| コミット＋プッシュ | `/commit-push` | 1分 |
| PR作成 | `/create-pr` | 1分 |
| ブランチ名提案 | `/branch-name-helper` | 30秒 |

### Issue実装

| 実装方法 | スキル | 確認 | 所要時間 | 推奨用途 |
|---------|-------|------|---------|---------|
| 手動確認 | `/implement-issue 123` | ✅ 各ステップ | 5-15分 | 重要な変更 |
| 完全自動 | `/auto-implement-issue 123` | ❌ なし | 3-10分 | 小規模な変更 |

## 使い分けガイド

### 🤔 どのスキルを使うべき？

#### シンプルな修正（1-2ファイル）

```
/commit-push
```

または

```
変更をコミットしてプッシュして
```

#### 小さなバグ修正（Issue対応）

```
/auto-implement-issue 45
```

#### 新機能追加（小〜中規模）

```
/implement-issue 123
```

#### 大規模なリファクタリング

```
/implement-issue 78
```

#### PR作成だけ

```
/create-pr
```

### 📊 ワークフロー選択チャート

```
変更の規模は？
├─ 小（1-2ファイル）
│  └─ Issue番号ある？
│     ├─ Yes → /auto-implement-issue <番号>
│     └─ No  → /commit-push
│
├─ 中（3-10ファイル）
│  └─ Issue番号ある？
│     ├─ Yes → /implement-issue <番号>
│     └─ No  → /branch-name-helper → /commit-push → /create-pr
│
└─ 大（10ファイル以上）
   └─ /implement-issue <番号>（各ステップで確認）
```

## 自然な会話での使い方

スラッシュコマンドを使わなくても、自然な会話でスキルを起動できます：

### Git操作

```
変更をコミットして
リモートにプッシュして
コミットしてプッシュして
PRを作成して
ブランチ名を提案して
前回の作業を再開したい
```

### Issue操作

```
Issue #123 を実装して
Issue 45 のバグを修正して
Issue #78 を完全自動で実装して
Issue 123 をクローズして
```

## トラブルシューティング

### スキルが認識されない

Claude Code を再起動してください。

### GitHub CLIが未認証

```bash
gh auth login
```

### コミットメッセージが気に入らない

生成されたメッセージは実行前に確認できます。修正を依頼するか、手動で `git commit` してください。

### 自動実装が失敗した

`/implement-issue <番号>` で手動確認しながら実装するか、手動で修正してください。

### PR作成時にエラー

ブランチがプッシュされているか確認：

```bash
git status
git push
```

### Issue番号がわからない

```bash
gh issue list
```

## よくある質問

### Q: スラッシュコマンドと自然な会話、どっちがいい？

A: どちらでもOKです。明確に実行したい場合はスラッシュコマンド、会話の流れで使いたい場合は自然な会話を使ってください。

### Q: コミットメッセージは編集できる？

A: はい。生成されたメッセージは実行前に確認・修正できます。

### Q: PRマージしたらIssueは自動でクローズされる？

A: はい。コミットメッセージとPR本文に `Closes #123` が含まれているため、PRマージ時に自動クローズされます。

### Q: エラーが出た場合は？

A: `/auto-implement-issue`（完全自動）は最大3回まで自動修正を試みます。それでも失敗したら手動修正が必要です。

### Q: 複数のIssueを同時に実装できる？

A: いいえ。1つずつ実装してください。並列で作業する場合は別々のブランチで行ってください。

### Q: 次のセッションでもスキルは使える？

A: はい。スキルは `~/.claude/commands/` に保存されているため、いつでも使えます。
