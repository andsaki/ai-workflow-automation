# resume-session

過去のセッション履歴を確認し、前回の作業内容を把握して続きを開始します。

## 実行手順

### 1. セッション履歴の確認

Codex の resume 機能を使用してセッション一覧を表示：

```bash
codex resume
```

最新のセッションを自動的に再開する場合：

```bash
codex resume --last
```

### 2. Git状態の確認

**Git状態の確認**:
```bash
git status --short
```

現在のブランチ：
```bash
git branch --show-current
```

最近のコミット：
```bash
git log --oneline -n 5
```

### 3. /tmp/ 内のバックアップファイルを確認

作業中に保存されたバックアップファイルや実験的なコードを確認：

**/tmp/ 内の開発関連ファイルを探す**:
```bash
ls -lt /tmp/*.{tsx,ts,vue,css,js,py,jsx,json} 2>/dev/null | head -10
```

**最近変更されたファイルを詳細に確認**:
```bash
find /tmp -maxdepth 1 -type f \( -name "*.tsx" -o -name "*.ts" -o -name "*.vue" -o -name "*.css" -o -name "*.js" -o -name "*.py" -o -name "*.jsx" \) -mtime -7 -exec ls -lh {} \; 2>/dev/null
```

**バックアップファイルの内容確認**:
- css-version.tsx などの命名パターンを探す
- プロジェクト名や機能名が含まれるファイル
- revert や backup といったキーワードを含むファイル

見つかったファイルは内容を確認し、以下を判断：
- 前回の作業で作成されたバックアップか
- revert された実装の保存版か
- 実験的な実装の試行版か
- 復元が必要なコードか

### 4. 作業コンテキストの復元

前回のセッションで行っていた作業を要約：

**確認項目**:
1. 実装中だった機能
2. 修正中だったバグ
3. 未完了のタスク
4. 次に行う予定だった作業

**関連ファイルの確認**:
最後に編集したファイルがあれば、その内容を確認：

```bash
git diff HEAD~1 HEAD --name-only
```

### 5. 作業の再開提案

前回のセッション内容を基に、以下を提案：

**再開オプション**:
1. **前回の続きを開始** - 未完了タスクから再開
2. **新しいタスクを開始** - 前回とは別の作業を開始
3. **前回の作業をレビュー** - 変更内容を確認してから判断

**提案例**:
```
前回のセッションでは「Panda CSSのデザインシステム実装」を行っていました。

未完了のタスク:
- [ ] Buttonコンポーネントの実装
- [ ] Cardコンポーネントのレスポンシブ対応

前回の最後に編集したファイル:
- components/design-system/Button.vue
- panda-config/recipes/button.ts

/tmp/ 内のバックアップファイル:
- /tmp/css-version.tsx (CSS実装版のバックアップ - revert済み)

続きから始めますか？それとも別の作業を始めますか？
```

### 6. 次のアクションを確認

ユーザーに次のアクションを確認：

1. 前回の続きを再開する
2. 別の作業を開始する
3. 変更内容をレビューしてから判断する
4. Git状態を整理してから再開する

---

## 分析のポイント

### セッション履歴から読み取る情報

**Git状態**:
- ステージングされた変更
- コミットされていない変更
- 現在のブランチ

**バックアップファイル**:
- /tmp/ 内の開発関連ファイル
- revert された実装の保存版
- 実験的なコードの試行版
- 代替実装のバックアップ

### 作業の優先順位

1. **未完了タスクが明確** → すぐに再開
2. **エラーで中断** → エラー解決から再開
3. **レビュー待ち** → 変更内容を確認
4. **コミット前** → コミットしてから次へ

---

## 注意事項

- セッション履歴は Codex が自動的に管理します
- 作業を再開する前に、必ずGit状態を確認してください
- 前回の変更がコミットされていない場合は注意が必要です
- セッション間で環境が変わっている可能性があります（依存関係の更新など）
